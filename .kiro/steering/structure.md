# Project Structure

## Top-Level Layout

```
paddelbuch/
├── _config.yml              # Main Jekyll config
├── _config_de.yml           # German locale build override
├── _config_en.yml           # English locale build override
├── _config_prefetch.yml     # Contentful pre-fetch only override
├── Gemfile                  # Ruby dependencies
├── Rakefile                 # Build pipeline tasks
├── package.json             # Node dependencies & scripts
├── amplify.yml              # AWS Amplify CI/CD config
├── index.html               # Homepage
├── 404.html                 # Error page
```

## Key Directories

| Directory | Purpose |
|---|---|
| `_plugins/` | Custom Jekyll plugins (Ruby). ~20 plugins handling CMS sync, API generation, tile generation, collection generation, etc. |
| `_data/` | YAML data files populated from Contentful during build. Contains `spots.yml`, `waterways.yml`, `obstacles.yml`, `notices.yml`, `protected_areas.yml`, `static_pages.yml`, and `types/` subdirectory for dimension tables. |
| `_layouts/` | Jekyll page templates: `default`, `spot`, `waterway`, `obstacle`, `notice`, `page` |
| `_includes/` | Reusable HTML/Liquid partials: map init, popups, detail content, header/footer, filter panel |
| `_i18n/` | Translation files: `de.yml` (German, default), `en.yml` (English) |
| `_sass/` | SCSS organised into `settings/`, `util/`, `components/`, `pages/` |
| `assets/` | Static assets: `js/` (browser JS modules), `css/`, `images/`, `fonts/` |
| `scripts/` | Utility scripts: vendor asset copying, font downloading, geometry processing (Python/Ruby/Node) |
| `spec/` | Ruby tests (RSpec + Rantly property-based tests) |
| `_tests/` | JavaScript tests (Jest + fast-check): `unit/`, `property/`, `integration/` |
| `deploy/` | CloudFormation template for AWS Amplify deployment |
| `docs/` | Project documentation: architecture, plugins, frontend, content model, testing |

## Generated Directories (not in source)

| Directory | Purpose |
|---|---|
| `_spots/`, `_waterways/`, `_obstacles/`, `_notices/`, `_static_pages/` | Jekyll collections generated by `collection_generator.rb` during build |
| `api/` | JSON API files generated by `api_generator.rb` |
| `_site/` | Final merged build output |
| `_site_de/`, `_site_en/` | Temporary per-locale build outputs (cleaned up after merge) |

## Plugin Architecture

Plugins in `_plugins/` are Jekyll generators/hooks with declared priorities. Key plugins:

- `contentful_fetcher.rb` (priority: highest) — orchestrates CMS data sync
- `sync_checker.rb` — Contentful Sync API delta detection
- `batch_fetcher.rb` — batched entry fetching for delta sync
- `contentful_mappers.rb` — transforms Contentful entries → YAML
- `collection_generator.rb` — generates Jekyll collection pages from `_data/`
- `api_generator.rb` — generates JSON API endpoints
- `tile_generator.rb` — spatial tile generation for map clustering
- `env_loader.rb` — loads `.env` files based on `JEKYLL_ENV`
- `cache_metadata.rb` — persists sync state between builds

## Content Flow

```
Contentful → ContentfulFetcher → _data/*.yml → CollectionGenerator → _spots/, _waterways/, etc.
                                             → ApiGenerator → api/*.json
                                             → TileGenerator → _data/.tile_cache/
```
