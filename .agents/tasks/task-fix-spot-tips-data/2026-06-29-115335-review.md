# Spot tip type field name correction

The spot mapper in `contentful_mappers.rb` looked up `:spot_tips` when resolving spot tip type references, but the actual Contentful field ID is `spotTipType`, which the Ruby SDK snakifies to `:spot_tip_type`. This caused `extract_reference_slugs` to receive `nil` and return an empty array, silently dropping all spot tip data from the API output. The fix changes the single symbol in the mapper, updates the corresponding test fixture, and corrects the documentation table.

Watch for: Nothing blocking. This is a one-symbol fix that follows a pattern already established by a prior bug fix (commit 72dc317, same class of error in the obstacle mapper). The change is minimal and the test suite confirms correctness.

## High-level view

The fix is mechanical: a single symbol rename from `:spot_tips` to `:spot_tip_type` in the field resolution call. The project's Contentful integration uses a consistent naming convention where reference fields take the content type name as their field ID (e.g., `:spot_type`, `:paddle_craft_type`, `:data_source_type`). The old name `:spot_tips` was an outlier that didn't match this convention or the actual Contentful schema.

## Silent failure mode from field name mismatch

The `resolve_field` helper returns `nil` for any key not present in the fields hash, and `extract_reference_slugs` treats `nil` as an empty collection — so a misnamed field silently produces an empty array rather than raising an error. This is the same failure mode that affected obstacle-to-spot resolution before commit 72dc317. With the fix, the naming is internally consistent across all reference fields in `map_spot`:

```
:spot_type          -> spotType (single ref)
:paddle_craft_type  -> paddleCraftType (multi ref)
:spot_tip_type      -> spotTipType (multi ref)
:data_source_type   -> dataSourceType (single ref)
```

## Test coverage

Not tested: there's no integration-level assertion that a real Contentful entry with `spotTipType` references produces non-empty slugs — but that's a constraint of the sandbox environment (no Contentful credentials), not a gap in the fix.
